# Тестирование приложения

## Назначение документа

Данный файл описывает порядок проверки работоспособности приложения классификации текстовых и графических данных, реализованного в папке `project/`.

## Цели тестирования

В ходе тестирования подтверждаются следующие свойства системы:

- корректная работа сервисного слоя классификации;
- корректная работа REST API;
- наличие и создание обученных моделей и отчетных артефактов;
- готовность приложения к локальному запуску;
- устойчивость базовых пользовательских сценариев.

## Состав проверок

### 1. Автоматические тесты `pytest`

Автотесты расположены в каталоге `project/tests/` и покрывают ключевые части приложения.

Проверяемые сценарии:

1. Классификация технологического текста через сервисный слой.
2. Классификация изображения треугольника через сервисный слой.
3. Проверка наличия артефактов обучения: моделей и графиков.
4. Проверка API-метода `POST /api/text/classify`.
5. Проверка API-метода `POST /api/image/classify`.
6. Проверка API-метода `GET /api/history`.

Используемые файлы:

- `project/tests/test_service.py`
- `project/tests/test_api.py`
- `project/tests/conftest.py`

### 2. Smoke-проверка приложения

После запуска приложения дополнительно выполняется запрос к служебному маршруту:

- `GET /api/health`

Проверка подтверждает, что приложение стартовало корректно и модели готовы к работе.

### 3. Ручная функциональная проверка

Дополнительно рекомендуется вручную проверить интерфейс `Gradio` по адресу:

- `http://127.0.0.1:8000/ui`

Минимальный набор ручных действий:

1. Ввести текст и получить класс с вероятностью.
2. Загрузить изображение фигуры и получить предсказание.
3. Открыть историю запросов.
4. Проверить отображение метрик моделей.

## Команды запуска проверок

Все команды выполняются из папки `project/`.

### Запуск приложения

```powershell
python main.py
```

Если используется текущее локальное окружение в корне ВКР:

```powershell
..\.venv\Scripts\python main.py
```

### Запуск автотестов

```powershell
python -m pytest --cov=src/vkr_classifier --cov-report=term-missing
```

Для текущего окружения:

```powershell
..\.venv\Scripts\python -m pytest --cov=src/vkr_classifier --cov-report=term-missing
```
